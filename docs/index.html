cx

A portable C++ library with zero external dependencies.

Builds on everything from your M3 Mac to a 1993 SPARCstation. Strings, containers, file I/O, JSON, networking, terminal handling, and more—all implemented from scratch. No Boost. No STL. No autoconf. No cmake. Just make.

The Problem

Most C++ libraries assume the C++ Standard Library exists and works correctly. That's fine for modern systems, but it breaks down fast on vintage Unix machines, embedded systems, or anything where the C++ runtime is incomplete or missing.

No Standard Library

No std::string, std::vector, std::map. Everything is implemented from scratch so it compiles everywhere.

Minimal Templates

Only for containers (CxSList<T>, CxHashmap<K,V>, CxHandle<T>). No template metaprogramming.

No Modern C++ Required

No auto, nullptr, lambdas, range-for, or constexpr. Compiles with GCC 2.8.1 from 1998.

Platform Isolation

OS-specific code is behind #ifdef guards. The API is identical on every platform, from macOS to SunOS 4.1.

What's Included

base Strings, containers, files, buffers, exceptions, reference counting
editbuffer Text editing buffer with cursor movement, insert/delete, UTF-8
screen Terminal control: cursor, colors (ANSI and 24-bit RGB), resize
keyboard Raw keyboard input with escape sequence parsing
json JSON parsing and generation without external libraries
net TCP/IP sockets with platform abstraction
thread Pthreads wrapper: threads, mutexes, condition variables
process Process spawning and output capture
buildoutput Compiler output parsing (GCC/Clang error formats)
commandcompleter Tab-completion engine with hierarchical completion
regex POSIX regex wrapper (Linux/macOS)
log Structured logging with configurable output
expression Mathematical expression parser and evaluator
tz Timezone handling over embedded CCTZ (Google, Apache 2.0)
b64 Base64 encoding and decoding

API Highlights

Strings

CxString s = "hello";
s += " world";
s.append("!");
int pos = s.firstChar('w');        // 6
CxString sub = s.subString(0, 5);  // "hello"
CxString upper = CxString::toUpper(s);

// UTF-8 with grapheme cluster support
CxUTFString utf;
utf.fromBytes(utf8_data, len, 4);  // 4-space tabs
int chars = utf.charCount();       // grapheme clusters
int width = utf.displayWidth();    // screen columns

Containers

// Linked list
CxSList<int> numbers;
numbers.append(1);
numbers.append(2);
for (CxSList<int>::Iterator it = numbers.begin();
     it != numbers.end(); ++it) {
    printf("%d\n", *it);
}

// Hash map
CxHashmap<CxString, int> ages;
ages.insert("alice", 30);
int *age = ages.find("alice");

JSON

// Parse
CxJSONBase *root = CxJSONFactory::parse(
    "{\"name\": \"cx\"}");
CxJSONObject *obj = (CxJSONObject *)root;
CxJSONString *name =
    (CxJSONString *)obj->getMember("name");

// Generate
CxJSONObject *config = new CxJSONObject();
config->addMember("version",
    new CxJSONNumber(1.0));
CxString json = config->toJsonString();

Terminal

CxScreen screen;
screen.openAlternateScreen();
CxScreen::placeCursor(10, 20);
CxScreen::setForegroundColor(
    new CxRGBColor(255, 100, 50));
screen.writeText("Hello in orange");
CxScreen::resetColors();

// Networking
CxSocket sock;
CxInAddress addr("example.com", 80);
sock.connect(&addr);
sock.send("GET / HTTP/1.0\r\n\r\n");
CxString response = sock.receive();

Quick Start

# Clone and build
mkdir -p ~/dev/cx && cd ~/dev/cx
git clone https://github.com/toddvernon/cx.git
cd cx && make

# Link against cx in your makefile
LIBS = -L../lib/darwin_arm64 -lcx_base -lcx_screen -lcx_json
INC  = -I../../..

# That's it. No configure scripts,
# no dependency resolution, no package managers.

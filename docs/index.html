The Problem

Most C++ libraries assume the C++ Standard Library exists and works correctly. That's fine until you need to support vintage Unix, embedded systems, or anything where the C++ runtime is incomplete.

cx provides a complete foundation: strings, containers, file I/O, JSON, networking, terminal handling. All implemented from scratch. The same source compiles unchanged on systems from 1993 to today.

No Boost. No STL. No autoconf. No cmake. No package managers.
Just make.

Why cx?

Zero Dependencies

Everything implemented from scratch. No external libraries. No package managers. Builds anywhere you have a C++ compiler.

No Standard Library

No std::string, std::vector, std::map. cx provides its own strings and containers that work on systems where the C++ runtime is broken or missing.

1993 to Today

The same source compiles on your M3 Mac and on SunOS 4.1.x, IRIX 6.5, and Solaris workstations from the early 1990s.

Platform Adaptive

Modern features on modern systems: UTF-8 with grapheme clusters, 24-bit RGB colors, threading. Graceful degradation on vintage systems.

Minimal Templates

Templates only where essential: containers and smart pointers. No template metaprogramming. Compiles with GCC 2.8.1.

Just Make

No autoconf scripts. No cmake configuration. No dependency resolution. Clone, run make, done.

Simple API

cx provides familiar APIs without external dependencies. Strings, containers, JSON, networking - all work the way you'd expect.

Strings & Containers

// Strings work like std::string
CxString name = "cx";
name += " library";
int pos = name.firstChar('l');

// UTF-8 with grapheme clusters
CxUTFString utf;
utf.fromBytes(data, len, 4);
int width = utf.displayWidth();

// Type-safe linked list
CxSList<CxString> files;
files.append("main.cpp");

// Hash map
CxHashmap<CxString, int> ages;
ages.insert("alice", 30);

JSON & Terminal

// Parse JSON
CxJSONBase *root =
  CxJSONFactory::parse(json_text);
CxJSONObject *obj =
  (CxJSONObject *)root;

// Generate JSON
CxJSONObject *cfg = new CxJSONObject();
cfg->addMember("name",
  new CxJSONString("cx"));

// Terminal with RGB color
CxScreen::setForegroundColor(
  new CxRGBColor(255, 100, 50));
screen.writeText("Hello");

Design Philosophy

📦

Self-Contained

Everything you need is in the box. No hunting for compatible versions of external libraries. No dependency conflicts.

🔧

Minimal C++

No auto, nullptr, lambdas, or range-for. Code compiles with compilers from 1998. Templates only for containers.

🌍

Portable First

Platform-specific code is isolated behind #ifdef guards. The API is identical everywhere. Write once, build anywhere.

Quick Start

# Clone
mkdir -p ~/dev/cx && cd ~/dev/cx
git clone https://github.com/toddvernon/cx.git

# Build
cd cx && make

# Libraries are built to ../lib/<platform>_<arch>/
ls ../lib/darwin_arm64/*.a
